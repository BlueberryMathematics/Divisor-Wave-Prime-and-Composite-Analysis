# Mathematical Formula Generation Systems
## Comprehensive Guide to Formula Creation in divisor-wave-python

The divisor-wave-python project provides multiple sophisticated systems for generating, analyzing, and working with mathematical formulas. This document covers all available formula generation capabilities.

---

## 🎯 **Overview of Formula Generation Systems**

### 1. **LaTeX Function Builder System** 📝
**Location**: `src/core/latex_function_builder.py`

The LaTeX Function Builder allows users to create custom mathematical functions using LaTeX notation and converts them to executable Python code.

#### Key Features:
- **LaTeX Input**: Write mathematical expressions in standard LaTeX notation
- **Automatic Conversion**: Converts LaTeX to executable NumPy functions
- **Function Library**: Built-in library of mathematical functions and constants
- **Error Handling**: Comprehensive error checking and validation
- **Plotting Integration**: Direct integration with plotting systems

#### Basic Usage:
```python
from src.core.latex_function_builder import LaTeXFunctionBuilder

builder = LaTeXFunctionBuilder()

# Create a function from LaTeX
latex_input = r"\sum_{n=1}^{\infty} \frac{1}{n^s}"
function_data = builder.create_function_from_latex(
    latex_input, 
    name="riemann_zeta_style"
)

# Use the generated function
result = function_data['python_function'](2.0)  # Compute ζ(2)
print(f"ζ(2) ≈ {result}")  # Should be close to π²/6
```

#### Advanced Examples:
```python
# Complex infinite product
latex_product = r"\prod_{n=1}^{\infty} \left(1 + \frac{x}{n^2}\right)"
product_func = builder.create_function_from_latex(
    latex_product, 
    name="custom_infinite_product"
)

# Parametric functions
parametric_latex = r"\frac{\sin(\pi x)}{x} \cdot e^{-x^2/\sigma^2}"
parametric_func = builder.create_function_from_latex(
    parametric_latex,
    name="windowed_sinc",
    parameters=['sigma']
)
```

### 2. **Mathematical Function Generator** 🔬
**Location**: `src/core/mathematical_function_generator.py`

Systematic function creation system that generates new mathematical functions by combining existing components using established mathematical rules.

#### Key Features:
- **Systematic Generation**: Rule-based function creation
- **Infinite Series Focus**: Specializes in infinite series generation
- **Component Combination**: Combines existing mathematical components
- **Pattern Recognition**: Identifies mathematical patterns and structures
- **Validation System**: Ensures mathematical validity of generated functions

#### Usage Example:
```python
from src.core.mathematical_function_generator import MathematicalFunctionGenerator

generator = MathematicalFunctionGenerator()

# Generate new infinite series variations
new_functions = generator.generate_series_variations(
    base_pattern="riemann_zeta",
    num_variations=5,
    complexity_level="medium"
)

for func in new_functions:
    print(f"Generated: {func['latex_formula']}")
    print(f"Description: {func['description']}")
```

#### Generation Categories:
- **Infinite Series**: Riemann-type, Dirichlet-type, Lambert series
- **Infinite Products**: Euler-type, Viète-type, custom products  
- **Special Functions**: Gamma function variants, hypergeometric series
- **Modular Forms**: Eisenstein series, theta functions
- **Number Theory**: L-functions, zeta function generalizations

### 3. **Function Registry System** 📋
**Location**: `src/core/function_registry.py`

Unified registry system that manages all mathematical functions with automatic LaTeX ↔ NumPy conversion and metadata management.

#### Key Features:
- **Centralized Management**: Single source of truth for all functions
- **Automatic Synchronization**: Keeps all components in sync
- **Bidirectional Conversion**: LaTeX ↔ NumPy automatic conversion
- **Metadata Management**: Function categories, descriptions, properties
- **Version Control**: Function versioning and migration support
- **Plugin Architecture**: Extensible system for custom functions

#### Usage:
```python
from src.core.function_registry import FunctionRegistry

registry = FunctionRegistry()

# Add new function to registry
registry.add_function(
    name="custom_zeta_variant",
    latex_formula=r"\sum_{n=1}^{\infty} \frac{(-1)^n}{n^s}",
    description="Alternating zeta function",
    category="special_functions",
    parameters=["s"]
)

# Auto-generates NumPy implementation and updates all systems
func = registry.get_function("custom_zeta_variant")
result = func.evaluate(s=2.0)
```

### 5. **LaTeX ↔ NumPy Conversion System** 🔄
**Location**: `src/core/latex_to_numpy_converter.py` & `src/core/python_to_latex_converter.py`

Bidirectional conversion system between LaTeX mathematical notation and executable NumPy code.

#### LaTeX → NumPy:
```python
from src.core.latex_to_numpy_converter import LatexToNumpyConverter

converter = LatexToNumpyConverter()

# Convert LaTeX to NumPy
latex_expr = r"\int_{0}^{\pi} \sin(x) \cos(x) dx"
numpy_code = converter.convert_latex_to_numpy(latex_expr)
print(numpy_code)
# Output: "integrate.quad(lambda x: np.sin(x) * np.cos(x), 0, np.pi)[0]"
```

#### NumPy → LaTeX:
```python
from src.core.python_to_latex_converter import PythonToLatexConverter

converter = PythonToLatexConverter()

# Convert Python function to LaTeX
def custom_function(x, n):
    return np.sum([1/k**n for k in range(1, int(x)+1)])

latex_repr = converter.python_to_latex(custom_function)
print(latex_repr)
# Output: r"\sum_{k=1}^{x} \frac{1}{k^n}"
```

## 🚀 **Advanced Workflows**

### Workflow 1: Complete Function Development Pipeline
```python
# Step 1: Create function from LaTeX
builder = LaTeXFunctionBuilder()
func_data = builder.create_function_from_latex(
    r"\prod_{n=1}^{\infty} \left(1 + \frac{x}{n}\right) e^{-x/n}",
    name="custom_infinite_product"
)

# Step 2: Add to registry
registry = FunctionRegistry()
registry.add_function_data(func_data)

# Step 3: Generate variations
generator = MathematicalFunctionGenerator()
variations = generator.generate_variations_from_base(func_data)

# Step 4: Validate and test
for variation in variations:
    # Test convergence, mathematical properties, etc.
    is_valid = registry.validate_function(variation)
    if is_valid:
        registry.add_function_data(variation)
```

### Workflow 2: AI-Assisted Discovery Integration
```python
# AI agents discover new patterns
ai_agent = MathematicalDiscoveryAgent()
discoveries = ai_agent.discover_new_patterns(
    domain="infinite_products",
    base_functions=registry.get_functions_by_category("products")
)

# Automatically integrate discoveries
for discovery in discoveries:
    # Convert AI representation to standard format
    func_data = ai_agent.format_discovery(discovery)
    
    # Validate mathematical properties
    if registry.validate_mathematical_properties(func_data):
        registry.add_function_data(func_data)
        
        # Generate LaTeX representation
        latex_repr = registry.get_latex_representation(func_data['name'])
        print(f"New discovery: {latex_repr}")
```
